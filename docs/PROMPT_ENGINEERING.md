# 提示词工程规范 (Prompt Engineering Guidelines)

> **版本**: v2.0  
> **适用模型**: DeepSeek-V3, GPT-4o, 通义千问, 豆包  
> **更新日期**: 2026-05

---

## 一、核心设计原则 (7 大原则)

### 1. 角色锚定 (Role Anchoring)
**目的**：让模型进入特定专业身份，激活相关知识

```yaml
# ❌ 错误示范
"你是一个 AI 助手，请帮助学生"

# ✅ 正确示范
"你是 TuringMate 团队的资深 408 考研辅导老师，图灵老师。
 专长：苏格拉底式启发教学、数据结构算法、操作系统、计算机网络、计算机组成原理。
 教龄：10 年，已帮助 500+ 考生上岸 985/211。"
```

**收益**：
- 激活相关领域的专业语料
- 限制模型越界回答
- 提升回答一致性和专业性

---

### 2. 任务结构化 (Task Decomposition)
**目的**：将复杂任务拆解为可执行步骤

```yaml
# 模板
# 任务
你的具体任务是：
1. [步骤 1] - <具体动作>
2. [步骤 2] - <具体动作>
3. [步骤 3] - <具体动作>

# 关键约束
- 必须：[硬性要求]
- 禁止：[明确禁止]
- 应当：[推荐做法]
```

---

### 3. Few-shot 示例 (In-Context Learning)
**目的**：通过示例让模型理解期望的输出模式

**示例数量建议**：
- 简单任务：1-2 个
- 复杂任务：3-5 个
- **示例要**包含"正面案例"和"反面案例"

**示例结构**：
```
---
**示例 1 - [场景]**
输入：[用户输入]
你的回复：
## 思考过程
[1-2 句分析]
## 输出
[期望输出]
---
```

---

### 4. 思维链 (Chain-of-Thought, CoT)
**目的**：强制模型先思考再回答，提升推理质量

```yaml
# 在 system prompt 中显式要求
## 思考过程
<先用 1-2 句话在心中分析学生的理解程度，识别误区>
## 教学行动
<提问或反馈>
```

**DeepSeek 模型特别支持**：会自动生成 `<think>...</think>` 块

---

### 5. 输出约束 (Format Constraints)
**目的**：确保输出可被程序解析、风格统一

```yaml
# JSON 输出约束
# 输出格式（严格遵守 JSON）
```json
{
  "field_name": "value",
  "enum_field": "option1|option2|option3"
}
```

# Markdown 输出约束
## 输出格式
## 标题 1
正文...

### 标题 2
- 列表项 1
- 列表项 2
```

---

### 6. 失败回退 (Failure Recovery)
**目的**：定义边界情况和降级策略

```yaml
# 注意事项
- 如果学生情绪激动 → 先共情再教学
- 如果学生重复问同一个问题 → 换新角度
- 如果学生问超出范围 → 礼貌拒绝并引导回主流程
- 如果识别失败 → 返回友好提示，不要胡编
```

---

### 7. 元认知 (Meta-Cognition)
**目的**：让模型自检答案质量

```yaml
# 输出前自检
- [ ] 我的回答是否给出了直接答案？(应该是"否")
- [ ] 我的回答是否包含鼓励性语言？
- [ ] 我是否遵守了字数限制？
- [ ] 我的回答是否针对学生的具体错误？
```

## 二、提示词结构模板

### 标准模板（适用于所有 Agent）

```markdown
# 角色定义
你是 [身份]，专长 [领域 1]、[领域 2]。
教龄/经验：[背景信息]。

# 核心行为准则（必须严格遵守）
1. [规则 1] - [具体解释]
2. [规则 2] - [具体解释]
3. [规则 3] - [具体解释]

# 当前任务
[任务背景和数据]
- 输入 1: {variable_1}
- 输入 2: {variable_2}

# 输出格式（严格遵守）
## 思考过程
<先分析>
## 教学行动
<主要输出>
## 预期反应
<期望的下一步>

# 关键约束
- **禁止**: [行为 A]
- **禁止**: [行为 B]
- **应当**: [推荐做法]

# 输出示例（Few-shot）
[2-3 个高质量示例]

# 开始回复
```

## 三、场景化提示词库

### 1. 苏格拉底导师（主对话）

**核心提示词**：[socratic_tutor_prompt](../backend/app/core/prompts_socratic.py)

**关键设计**：
- 5 个教学阶段（question → thinking → guidance → practice → review）
- 3 个 Few-shot 覆盖典型场景（学生困惑、学生答对、学生答错）
- 明确"绝不直接给答案"的硬约束

---

### 2. 情绪感知模式

**触发条件**：检测到学生情绪异常（通过 sentiment analysis）

**核心提示词**：[EMOTION_AWARE_OVERRIDE](../backend/app/core/prompts_socratic.py)

**情绪 → 策略映射**：

| 情绪 | 检测特征 | 应对策略 |
|-----|---------|---------|
| frustrated | 反复说"不会"、"看不懂" | 先共情再教学，降低难度 |
| confused | 多轮无进展、问同一问题 | 换类比、图形化方式 |
| anxious | "来不及了"、"完了" | 强调"基础已掌握" |
| confident | 答对率高、主动延伸 | 适当挑战，引入新视角 |
| bored | 答题过快、敷衍 | 提高难度 |
| engaged | 主动追问、深度思考 | 顺势深入 |

---

### 3. 拍照识别

**核心提示词**：[PHOTO_RECOGNITION_PROMPT](../backend/app/core/prompts_socratic.py)

**关键设计**：
- 多模态结构化输出（LaTeX + Markdown）
- OCR 置信度评估
- 历年真题关联

---

### 4. 薄弱点诊断

**核心提示词**：[DIAGNOSIS_PROMPT](../backend/app/core/prompts_socratic.py)

**关键设计**：
- JSON 结构化输出
- 严重程度量化（1-5）
- 跨科关联分析
- 可执行补救建议

---

### 5. 举一反三

**核心提示词**：[VARIANT_GENERATION_PROMPT](../backend/app/core/prompts_socratic.py)

**关键设计**：
- 难度递进策略
- 答案唯一性约束
- 配套解析和常见错误

## 四、模型适配策略

| 模型 | 特性 | 提示词调整建议 |
|-----|------|---------------|
| **DeepSeek-V3** | 中文强、CoT 友好、便宜 | 偏好 Markdown，可放心使用 `<think>` 块 |
| **GPT-4o** | 多模态强、英文好 | 需要明确指示中文输出，避免中英混杂 |
| **通义千问** | 平衡性好 | 标准提示词即可 |
| **豆包 Doubao** | 国产、价格低 | 简短直接，结构化指令效果更好 |

## 五、A/B 测试与版本管理

### 版本命名规范
```
prompt_<场景>_v<主版本>.<次版本>
例：socratic_tutor_v2.3
```

### A/B 测试流程
1. 准备 2 个版本的提示词（A、B）
2. 随机分配给 10% 流量
3. 收集关键指标：
   - **续答率** - 学生是否继续对话
   - **答对率** - 学生在引导下能否解出题
   - **平均轮次** - 完成一次教学对话需要的轮数
   - **满意度** - 学生主动评价
4. 至少运行 1 周后做显著性分析
5. 胜出版本推全

### 效果监控指标
- 提示词版本：`<git hash>` 或 `<semver>`
- 模型版本：`deepseek-v3-20250101`
- 温度参数：`0.7`
- 关键指标：续答率、答对率、平均轮次

## 六、安全与合规

### 禁止行为
1. **直接给答案** - 违反教学原则
2. **学术不端引导** - 替写代码、代做作业
3. **政治敏感内容** - 涉及政治立场时保持中立
4. **个人信息索取** - 不询问身份证号、密码等
5. **医疗/法律建议** - 超出 AI 能力范围

### 兜底话术
```
当用户问及敏感话题时：
"这个话题超出了我的辅导范围，我们还是回到 [数据结构/操作系统/...] 吧！"
```

## 七、提示词调试技巧

### 1. 简化测试
先把提示词压到 100 字以内，验证基本功能。

### 2. 定位失败
- 格式问题？→ 检查输出约束
- 内容问题？→ 检查 Few-shot 示例
- 推理问题？→ 强化 CoT

### 3. 调参建议
- `temperature=0.3-0.5`：教学场景，需要稳定
- `temperature=0.7-0.9`：创意场景（举一反三）
- `max_tokens`：根据长度限制设定

### 4. 常见陷阱
- ❌ "回答详细一些"（太模糊）
- ✅ "控制在 50-120 字"
- ❌ "做一个好老师"
- ✅ "使用苏格拉底式提问法"
- ❌ "输出 JSON"
- ✅ "输出格式（严格 JSON）：```json {...}```"
